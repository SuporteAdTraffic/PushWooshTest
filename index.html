Getting Started - Consumo das APIs de Relatórios AD

Conectando-se às APIs

As APIs seguem o padrão REST API e retornam os dados no formato JSON.
O acesso é feito via requisições HTTP POST e exige autenticação por Bearer Token.

Como obter o Bearer Token

Antes de consumir qualquer endpoint de dados, é necessário emitir um token de acesso.

Endpoint de autenticação:
POST https://adtraffic-holdingclube.adverhouse.com.br/api/v3/adtraffic/issuetoken

Parâmetros obrigatórios (no corpo da requisição):
- appId: bc6d2393
- appKey: 638a9f220197
- username: o login do usuário com permissões válidas
- userKey: 4f849e6e

Exemplo de corpo da requisição (JSON):
{
  "appId": "bc6d2393",
  "appKey": "638a9f220197",
  "username": "seu.usuario@empresa.com.br",
  "userKey": "4f849e6e"
}

A resposta será um token de acesso, usado nos próximos passos.

Como consumir as APIs

Com o token em mãos, você pode consumir os endpoints de dados, também via requisição POST.

Endpoints disponíveis:
- Listar Ordens de Compra:
  POST https://adtraffic-holdingclube.adverhouse.com.br/api/v3/adtraffic/ordemcompra/listar
- Listar Ordens de AP:
  POST https://adtraffic-holdingclube.adverhouse.com.br/api/v3/adtraffic/ap/listar

Requisição:
- Método: POST
- Headers obrigatórios:
  Authorization: Bearer [token_obtido]
  Content-Type: application/json
- Corpo da requisição:
  {}
